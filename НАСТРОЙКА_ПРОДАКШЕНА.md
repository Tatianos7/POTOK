# Настройка продакшена для корректной работы упражнений

## Проблема

На продакшене (GitHub Pages) категории упражнений не отображаются, хотя локально все работает.

## Причина

1. **Разные переменные окружения** - локально и на продакшене используются разные настройки Supabase
2. **Данные не импортированы** - в продакшн базе данных нет категорий и упражнений

## Решение

### 1. Настройка переменных окружения в GitHub

1. Откройте репозиторий на GitHub
2. Перейдите в **Settings** → **Secrets and variables** → **Actions**
3. Добавьте следующие секреты:
   - `VITE_SUPABASE_URL` - URL вашего Supabase проекта
   - `VITE_SUPABASE_ANON_KEY` - Anon key вашего Supabase проекта

### 2. Инициализация данных в Supabase

**Вариант А: Автоматическая инициализация (рекомендуется)**

Приложение автоматически создаст базовые категории при первом запуске, если их нет в базе.

**Вариант Б: Ручная инициализация через SQL**

1. Откройте Supabase Dashboard
2. Перейдите в **SQL Editor**
3. Выполните SQL скрипт из файла `supabase/workout_schema.sql`
   - Это создаст все таблицы, категории, мышцы и упражнения

### 3. Проверка

После настройки:
1. Дождитесь автоматического деплоя (или запустите вручную)
2. Откройте https://tatianos7.github.io/POTOK/
3. Войдите в приложение
4. Перейдите в "Дневник тренировок"
5. Нажмите "+" (Добавить упражнение)
6. Должны отобразиться все 7 категорий

## Автоматическая инициализация

Приложение автоматически:
- Проверяет наличие категорий при загрузке страницы тренировок
- Создает базовые категории, если их нет
- Выполняется только один раз (помечается в localStorage)

## Что делать, если категории все еще не отображаются

1. Проверьте консоль браузера на наличие ошибок
2. Убедитесь, что переменные окружения настроены в GitHub Secrets
3. Проверьте, что Supabase проект доступен и RLS политики разрешают чтение
4. Выполните SQL скрипт вручную в Supabase SQL Editor
