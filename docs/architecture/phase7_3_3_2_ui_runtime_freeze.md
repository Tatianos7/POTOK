# Phase 7.3.3.2 — UI Runtime & Design System Freeze

## Product Archetype
**Calm Power Coach** — гибрид заботы и силы.  
Поддержка без давления, уверенное ведение к результату.

---

## A) Design System Core (Care + Power)
**Цвета**
- **Trust:** мягкий синий
- **Calm:** нейтральный серо‑бежевый
- **Focus:** тёмный графит
- **Progress:** зелёный
- **Risk:** оранжевый / красный
- **Premium:** золото / янтарный

**Типографика**
- Спокойная, уверенная, без «фитнес‑крика».
- Заголовки — чёткие, короткие.
- Тексты — мягкие, поддерживающие.

**Иерархия**
- Сначала безопасность и ясность.
- Затем действие и мотивация.

**Состояния UI**
- normal / progress / fatigue / recovery / blocked / premium‑locked / success

---

## B) Screen Archetypes
### Goal & Measurements
- **Эмоциональная цель:** “я стартую с уверенностью”.
- **Психологическое состояние:** осторожный интерес.
- **Тон:** care‑heavy.
- **Trust & Safety:** объяснить, что данные нужны для безопасности.
- **Explainability:** почему цель и КБЖУ такие.
- **Premium‑ценность:** план после расчёта.
- **Error/Recovery:** мягкий retry, без вины.

### Food Diary (Manual Mode)
- **Эмоциональная цель:** “я контролирую питание”.
- **Тон:** care + контроль.
- **Trust & Safety:** мягкое отображение отклонений.
- **Explainability:** как дневник влияет на прогресс.
- **Premium‑ценность:** план‑сравнение.

### Today (Follow Plan Mode)
- **Эмоциональная цель:** “меня ведут, но я свободен”.
- **Тон:** balance care/power.
- **Trust & Safety:** safety‑first, без давления.
- **Explainability:** “почему сегодня так”.
- **Premium‑ценность:** весь экран — ценность премиума.

### My Program
- **Эмоциональная цель:** “я вижу свой путь”.
- **Тон:** power‑forward, но спокойный.
- **Trust & Safety:** объяснение адаптаций.
- **Explainability:** почему план менялся.

### Progress
- **Эмоциональная цель:** “я вижу рост”.
- **Тон:** мотивирующий, без давления.
- **Trust & Safety:** нелинейность прогресса.
- **Explainability:** почему тренд такой.

### Habits
- **Эмоциональная цель:** “я держу ритм”.
- **Тон:** care‑heavy.
- **Trust & Safety:** срыв = часть пути.
- **Explainability:** привычки влияют на результат.

### Profile & Subscription
- **Эмоциональная цель:** “я в безопасности и контроле”.
- **Тон:** neutral + clarity.
- **Trust & Safety:** прозрачность подписки и данных.
- **Explainability:** why paywall / why consent.

### Coach Layer
- **Эмоциональная цель:** “меня поддерживают”.
- **Тон:** эмпатия + уверенность.
- **Trust & Safety:** анти‑стыд, анти‑давление.
- **Explainability:** эмоциональная логика.

## C) UX State Machine (Global)
- Manual / Follow Plan
- Low Energy / High Motivation
- Relapse / Recovery
- Trust Drop / Confidence Drop
- Premium Gate
- Coach Intervention

---

## D) Freeze Contracts
**Замораживается как стандарт**
- Calm Power Coach tone
- Safety‑first UX (обязательный)
- Explainability hooks
- Premium как ценность (не барьер)
- Manual Mode как полноценный путь

**Основа для Phase 7.4**
- Единый визуальный язык
- Состояния и тоны коммуникации

**Нельзя менять без пересборки**
- Принцип safety‑first
- Структуру Premium‑ценности
- Explainability как обязательную поверхность
