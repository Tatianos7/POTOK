## Видение
Phase 8 переводит продукт из «программы» в живого долгосрочного компаньона здоровья.
Цель — платформа, которая поддерживает физический прогресс, психическую устойчивость и устойчивый образ жизни.
Архетип Calm Power Coach остаётся ядром: забота + внутренняя сила, без давления, доверие по умолчанию.

Phase 7 = Product  
Phase 8 = Living System

## Слои системы
1. **AI Coach Core** — постоянная личность, диалоговый движок и слой рекомендаций.
2. **Long‑Term User Model** — устойчивый профиль на месяцы/годы (физика + психология + поведение).
3. **Emotional Safety Layer** — анти‑стыд, анти‑давление, этические защитные рамки.
4. **Social / Community Layer** — будущий опциональный слой поддержки (opt‑in).
5. **Expert / Doctor / Coach Layer** — будущая эскалация к профессионалам.
6. **Devices & Sensors Layer** — Apple Health, часы, Vision Pro, весы.
7. **Life OS Layer** — цели за пределами фитнеса: стресс, сон, энергия, фокус.

## AI Coach Core
**Цель:** сохранять постоянную, безопасную, объяснимую коуч‑личность на длинных горизонтах.

**Основной архитектурный документ**
- `sprint_8_1_ai_coach_core.md`

**Возможности**
- Многошаговый диалог с контекстной памятью
- Переосмысление целей (физические + эмоциональные)
- Адаптивный тон на основе доверия и усталости
- Объяснимые рекомендации с уровнем уверенности
- Safety‑интервенции (перетрен, боль, рисковые паттерны)

**Режимы**
- Гайд для онбординга (краткосрочно)
- Ежедневный коуч (Today + habits)
- Рефлексивный коуч (Progress review)
- Коуч восстановления (relapse, pain, fatigue)

## Память и идентичность
**Цель:** помнить важное, забывать лишнее.

**Типы памяти**
- **Стабильная:** долгосрочные цели, предпочтения, мед‑ограничения, risk‑флаги
- **Сезонная:** текущие фазы, активные привычки, контекст программы
- **Эфемерная:** настроение дня, единичные события, короткие петли

**Правила забывания**
- Затухание краткосрочных эмоциональных всплесков
- Агрегация повторяющихся паттернов в устойчивые сигналы
- Удаление чувствительных деталей без явного закрепления

**Explainability памяти**
- «Мы помним это, потому что оно влияет на безопасность или долгосрочное направление».

## Эмоциональная безопасность
**Принципы**
- Без стыда
- Без давления
- Без наказаний
- Без guilt‑paywall

**Интервенции**
- Guard‑rail при боли и усталости
- Сообщения восстановления
- «Возвращение мягко» после срыва
- Эскалация в Expert Layer (в будущем)

## Долгосрочная мотивация
**Цель:** формировать привычки на годы, а не на недели.

**Механизмы**
- Долгосрочные вехи (3/6/12 месяцев)
- Поддержка малых побед
- Укрепление идентичности («ты становишься тем, кто…»)
- Фрейм «медленный прогресс — это тоже прогресс»

**Профилактика выгорания**
- Авто‑делоад‑советы
- Стрики с окнами прощения
- Празднование дней отдыха

## Эволюция доверия
**Доверие растёт, когда:**
- рекомендации последовательны и объяснимы
- ошибки данных прозрачны и восстанавливаемы
- пользователь чувствует контроль над решениями
- система помнит важное и уважает границы

**Доверие падает, когда:**
- рекомендации кажутся случайными
- paywall ощущается как давление
- теряются данные или появляется чувство оценки

**Восстановление доверия**
- явные объяснения
- возможность отката изменений плана
- recovery‑first коммуникация

## Расширение платформы
**Вектор**
- **Voice**: ежедневный коуч и микрочекины
- **Vision**: анализ техники, валидация замеров, реабилитационные подсказки
- **Doctors/Experts**: эскалация при safety‑критичных состояниях
- **Community**: opt‑in поддержка, анонимные бенчмарки
- **Sensors**: биометрика для усталости, сна, восстановления

**Guardrails**
- Явные согласия на чувствительные данные
- Никаких социальных функций без opt‑in
- Медицинские дисклеймеры везде

## E2E‑сценарии (200–240)
**200–210: долгосрочный ритм**
- Возврат после 2 недель паузы с мягким онбордингом
- Режим восстановления после боли
- Признание вех без давления

**211–220: доверие и объяснимость**
- «Почему план изменился» после сдвига данных
- «Почему стоит замедлиться» с safety‑обоснованием
- «Почему система советует отдых» с confidence‑флагом

**221–230: расширение платформы**
- Voice‑чек‑ин вместо push‑уведомления
- Падение сна по wearable → день восстановления
- Сессия Vision Pro обновляет индекс усталости

**231–240: social & expert**
- Анонимное сравнение без стыда
- Личное деление вехой
- Эскалация к эксперту при устойчивых мед‑флагах
