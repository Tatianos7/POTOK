# Phase 8 — Public MVP Gate (Release Readiness)

## Цель
Зафиксировать критерии готовности Phase 8 к публичному MVP‑релизу без изменения архитектуры и UX‑контрактов.

## 1. Emotional Safety
- Crisis‑mode активируется при distress.
- Никаких обвиняющих формулировок.
- Риск‑сценарии приводят к safe‑tone.

## 2. Coach Non‑Intrusiveness
- Есть лимит вмешательств.
- Есть режим «только при риске».
- Пользователь может выключить коуча.

## 3. Explainability Transparency
- Пользователь понимает «почему».
- Есть fallback «без деталей».
- История решений читаема и краткая.

## 4. Legal & Consent
- Дисклеймеры медицинской/психологической ответственности.
- Прозрачный контроль памяти.
- Возможность сбросить доверие/историю.

## 5. AI Behaviour Predictability
- Реакции повторяемы.
- Не меняется тон без причины.
- Доверие не обрушивается резко.

## 6. Market Readiness
- Продукт ощущается как поддержка, а не давление.
- Premium‑ценность ясна и этична.
- UX не ломается при деградации.

## 7. Ограничения
Не менять:
- E2E сценарии
- Memory schema
- Runtime pipeline
- UI‑Kit

## 8. Definition of Done
- Все пункты выполнены.
- Нет критических сбоев в рантайме.
- Есть отчёт о метриках (latency, memory hits, explainability usage).
